Step 02 · Enrich · four passes

Extract → Cluster → Score → Label.

Each pass runs once over the corpus and writes a new artifact. The next pass reads it.

PASS 01
Extract
Parse raw rows into a uniform insight skeleton. One extractor per source — the shape of a TripAdvisor row differs from a Reddit comment and from a YouTube transcript chunk.
INraw_*.json
OUTflat_insights_*.json (per source)
PASS 02
Cluster
Embed every insight with text-embedding-3-small. Run HDBSCAN to group semantically similar rows. Name each cluster with a canonical observation via an LLM pass over its centroid texts.
INflat_insights_all.json
OUT1,450 named clusters
PASS 03
Score
Assign each cluster a quality_tier (A, B, or C) computed from size, source diversity, and sentiment coherence. A claim corroborated across Reddit + TripAdvisor weighs more than the same claim repeated by twenty TA reviews.
INnamed clusters
OUT+ quality_tier · source_mix
PASS 04
Label
Tag every row with: L1/L2/L3 taxonomy, intent_tag (recommendation, warning, explanation, history, …), sub_attraction (named entity), numeric_anchors (extracted numbers + units).
INscored clusters
OUTflat_insights_enriched.json (22 fields)

Step 04 · MCP server

Six tools.

JSON-RPC over stdio for local agents. Flask HTTP wrapper for browser clients. Both call the same retrieval functions over the same in-memory store.

search_clusters(query, poi, intent_l1?, min_tier?)
Semantic cluster search. ~200 ms. Cosine over centroid embeddings + structured filters.
search_insights(query, poi, filters?)
Per-row search. Filters: sub_attraction, intent_tag, sentiment, has_numeric, rating range.
explain_cluster(cluster_id, max_quotes?)
Returns the cluster card plus N raw reviewer quotes with source_id for back-tracing.
poi_stats(poi)
Aggregate counts by tier, sentiment, intent_tag, L1, source. Returns derived verdict_score.
sub_attractions(poi, limit?)
Named sub-attractions grouped from insights. Median minutes spent, top intent tags, source diversity.
ask(query, poi, booking_context?)
Full RAG round-trip — agent loop wrapped as a single tool call.

Step 05 · ask_gems

The RAG loop.

Function-calling agent. Up to 6 iterations. Hard requirement: ≥ 2 corroborating quotes per claim or confidence is downgraded.

01
Query in
{ query, poi, booking_context? }
02
Retrieve
Agent calls search_clusters / search_insights with structured filters.
03
Cite
explain_cluster pulls raw quotes + source_id for the candidate clusters.
04
Answer out
{ answer, citations[], confidence } — structured output, no free prose.
CONSTRAINT
citations.length ≥ 2 OR confidence = "low"
CONSTRAINT
every citation carries source_id traceable to raw_*.json
MODEL
gpt-4o for synthesis · text-embedding-3-small for retrieval
LATENCY
2–4s typical · 8s p99
